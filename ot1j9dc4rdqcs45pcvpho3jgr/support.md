# Support
This project uses GitHub Issues to track bugs and feature requests.

## Report issues <!-- modified from elasticsearch -->
If you think you have found a bug, first make sure that you are using the correct and updated version of the repository as the issues may already have been fixed. If not, search the issues list on GitHub in case a similar issue has already been opened.

It is very helpful if you can prepare a reproduction of the bug. In other words, provide a small test case which we can run to confirm your bug. It makes it easier to find the problem and to fix it. Once you are ready to report an issue, visit [ticketing]()<!-- insert link -->

## Feature requests
If you find yourself wishing for a feature that doesn't yet exists, you are probably not alone. Search the issues list on GitHub in case others have already opened a similar request. If a similar request have already been submitted, it will help if you can vote on the requests to inform us of the demand for that feature.

If the feature have not been requested, visit [ticketing]()<!-- insert link --> to submit the request.
