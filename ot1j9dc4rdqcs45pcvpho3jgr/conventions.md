# Conventions

There are conventional styles, terms and terminologies used in this documentation.

| Convention | Format | Description |
| ---------- | ------ | ----------- |
|placeholder|`<placeholder_name>`|A value you must replace with your own specific data. Do not type the < > brackets. e.g. cd <file_path> should be interpreted as my_project/source.

## Type cases

| Name | Description | Example |
| ---- | ----------- | ------- |
| snake case | All characters in lower case with words separated with an underscore ("_") | snake_case |
| camel case | Except for the first word, the first letter of every word is capitalized with no spacing between words. For abbreviations, treat them as words (i.e. only first letter is capitalized, please see example) To be avoided if possible unless it is part of a pre-defined framework such as in Qt QML. | camelCase<br> myId (notice it is NOT myID) |
| pascal case | The first letter of every word is capitalized with no spacing between words | PascalCase<br> MyId (notice it is not MyID) |
| title case | The first letter of every word is capitalized with spacing between words | Title Case |
| sentence case | Only the first letter of the first word is capitalized with spacing between words | Sentence case |
 kebab case | Words are separated by a dash ("-") | kebab-case |
